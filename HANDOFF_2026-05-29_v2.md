## 1. このセッションでやったこと（すべて main にコミット済み: `e204cde` + 後続）

### エクスポート機能の完成（WebCodecs 経路）
- **Phase 3.3c**: テキスト演出（typewriter / wave / highlight-bar / outline-reveal / shadow-drop /
  kinetic 系）を WebCodecs で描画。`drawLayer` の comment ケースを `drawAnimatedTextFrame` 経路に接続。
  `drawAnimatedTextFrame` の no-char 分岐の textAlign バグも修正。
- **Phase 3.2**: Live2D character レイヤー対応。エクスポート開始時に `composeCharacterLayerVideo` で
  各キャラを VP9+alpha WebM に事前焼き → `buildVideoStream` で video と同じ VideoSampleSink 経路に流す。
- **吹き出し (bubble)**: `drawLayer` の comment が bubble 形状を描いていなかったのを `drawBubbleShape`
  共通ヘルパで対応（preview `BubbleSvg` と同じ `bubbleFullPath`）。

### 新アニメ（別エージェントの preview 実装 + 本セッションで export 実装）
- `grow-up` / `grow-down` / `grow-right` / `grow-left`（端から伸びる。`computeCanvasAnim` に
  scale 原点 `originX/originY` を追加して transform-origin を再現）
- `arc-sweep`（shape:"arc" 専用。`drawArcShape` で arcEnd を「1本のペン先 0°→360° linear」方式で
  時間補間。preview `ArcShapeSvg` と完全一致）

### プレビュー音声バグ修正
- 通常音量(0..1)は HTMLAudioElement 直結に戻し、100%超のときだけ Web Audio GainNode 経由に。
  （直前の変更で全レイヤーを GainNode 経由にした結果、AudioContext suspended で**無音**になっていた）

### #1 画質プリセット撤去
- WebCodecs はビットレート指定で画質を変えても速度がほぼ変わらない（所要時間は Canvas 合成が支配的）
  ため、画質 UI を撤去し常に `QUALITY_HIGH` 固定に。

### #2 ffmpeg 動画合成経路の物理削除
- **TS**: `src/lib/exportTemplate.ts` 削除 / `src/lib/video.ts` は `ProgressUpdate` 型のみに縮小。
- **Rust** (`src-tauri/src/lib.rs`, 約1670行削除): `compose_template_video` コマンド + `_inner` +
  filter 式ビルダ群 + キャンセル機構 (`ExportCancelState` / `run_ffmpeg_cancellable` / `cancel_export`) +
  `parse_ffmpeg_time` / `escape_movie_path`。`cargo check` 0 エラー / 0 警告。
- **保持した共有部分**: `save_render_chunk` / `save_final_video`（WebCodecs が使用）、
  ffmpeg.exe 呼び出し全般・`escape_xml`・TTS / BGM / `get_audio_duration`（**ffmpeg は音声系でまだ必要**）。

## 4. 落とし穴メモ

- **ビルド時はアプリを終了する**: 起動中だと release exe がロックされ `tauri build` が
  `os error 5 (アクセス拒否)` で失敗する。`Get-Process shorts-script-gen` で確認してからビルド。
- デスクトップショートカット `shorts-script-gen.lnk` は `src-tauri/target/release/shorts-script-gen.exe`
  を直接指す。`npm run tauri build` で更新すれば再インストール不要（要アプリ再起動）。
- `templates/*.json` は git 非管理（ユーザーテンプレ・絶対パス入り）。共有は pack(.zip) 経由。
- 別エージェントと同一ワーキングツリーを共有することがある。preview 側の変更が入ったら
  export 側（layerComposer / layerAnimCanvas）との数式一致を再確認すること（今回 arc-sweep で顕在化）。
