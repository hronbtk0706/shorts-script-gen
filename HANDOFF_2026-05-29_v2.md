## 1. このセッションでやったこと（すべて main にコミット済み: `e204cde` + 後続）

  kinetic 系）を WebCodecs で描画。`drawLayer` の comment ケースを `drawAnimatedTextFrame` 経路に接続。

## 2. 現在のアーキテクチャ（重要）

- **プレビュー**: React / CSS / HTMLAudioElement（編集画面のリアルタイム再生）
- **エクスポート**: WebCodecs 1 系統のみ（`exportTemplateWebCodecs.ts`）
  - 毎フレーム `renderLayersOnContext` → `drawLayer`（preview と同じ Canvas 合成）
  - 入退場/Ambient = `layerAnimCanvas.ts`、テキスト演出 = `drawAnimatedTextFrame`
  - 音声 = `mixAudioLayers`（OfflineAudioContext）、video = VideoSampleSink、character = 事前焼き
  - 保存 = 8MB チャンクで `save_final_video` invoke
- **ffmpeg は動画合成に使わない**（音声系のみ）。Rust に `compose_template_video` は無い。

**新レイヤー機能のチェックリストは [CLAUDE.md](CLAUDE.md) を参照**（WebCodecs 前提に更新済み）。

## 3. 残作業

### 🟢 掃除（機能影響なし）
1. **旧型残骸の削除**: `types.ts` に `Motion` / `SceneEffects` / `BodySegment` / `Script` 等が残存。
   「シーン・セグメント概念は廃止済み」と不一致。
2. 前版 `HANDOFF_2026-05-29.md` は ffmpeg 前提の記述が多く古い（この v2 が現状）。

### ⚠️ 原理的残差（Canvas 2D の制約。preview 専用差として許容 or 要判断）
- **A5 flip-in/out**: preview は真 3D(perspective)、export は scaleX 2D 近似（WebGL 化しないと不可）
- **E6 character 物理 dt**: preview は rAF 可変、export は固定 1/30（export の方が決定的で正）
- **C6 改行位置**: DOM vs Canvas measureText のフォントメトリクス差（稀に 1 文字ズレ）
- **C5/E7 comment box overflow**: テキストが箱を超える稀なケースで preview はクリップ / export はしない。
  必要なら `drawLayer` の comment に矩形 clip を追加すれば対応可。

### 📋 実機検証（推奨）
- 実装は preview と数式一致させているが、character のリップシンク/見た目、ambient 振幅、
  text effect、grow-*/arc-sweep、bubble は実機エクスポートでの目視確認が確実。

## 4. 落とし穴メモ

- **ビルド時はアプリを終了する**: 起動中だと release exe がロックされ `tauri build` が
  `os error 5 (アクセス拒否)` で失敗する。`Get-Process shorts-script-gen` で確認してからビルド。
- デスクトップショートカット `shorts-script-gen.lnk` は `src-tauri/target/release/shorts-script-gen.exe`
  を直接指す。`npm run tauri build` で更新すれば再インストール不要（要アプリ再起動）。
- `templates/*.json` は git 非管理（ユーザーテンプレ・絶対パス入り）。共有は pack(.zip) 経由。
- 別エージェントと同一ワーキングツリーを共有することがある。preview 側の変更が入ったら
  export 側（layerComposer / layerAnimCanvas）との数式一致を再確認すること（今回 arc-sweep で顕在化）。
