## 2. プレビューとエクスポートの mismatch 監視点（網羅調査）

CLAUDE.md にある「両系統を同時に変更する」鉄則を破ると以下が起きる。**全部はチェックしきれていない**。優先度高い順に列挙。

### 🔴 確定している既知の問題

| 問題 | 症状 | 関連ファイル |
|---|---|---|
| **コメント/テキスト box overflow** | プレビューは text が box からはみ出して表示。エクスポートは `applyShapeClip` で矩形に clip → クリップマーク（引用符）等の小さい box に大きな font が clip される | `layerComposer.ts:205` (`applyShapeClip` 呼び出し), `TemplateCanvas.tsx:462` (`overflow: hidden`) |
| **static + pop の overshoot 無効化** | プレビューは bouncy だが、エクスポートは `min(1)` でクランプ。pad で出力固定するための妥協 | `lib.rs:2310-` (`needs_size_stabilize`) |

### 🟡 未検証だが mismatch が出やすい箇所

| 項目 | 懸念 |
|---|---|
| **entry/exit anim**: `blur-in`/`blur-out` | プレビュー CSS `filter: blur` vs エクスポート `gblur` または `fade` で代替。**時間依存 gblur が ffmpeg で表現困難** で `fade` 代用と引き継ぎに書かれていた |
| **entry/exit anim**: `elastic-pop` / `flip-in` / `stretch-in` / `roll-in` | プレビューは CSS transform でフル表現。エクスポートも scale_anim_expr で対応しているが、static の場合 `min(1)` クランプの影響で挙動が浅くなる |
| **ambient**: `pulse` (scale 系) | static layer で `pad` をかけたぶん、出力サイズは固定だが pulse のサイズ変動はクランプ範囲外で発生する可能性。要確認 |
| **ambient**: `glow-pulse` / `rainbow` / `neon` | プレビューは CSS filter / box-shadow / gradient で実装。エクスポートは `hue` や `geq` で近似。**完全一致しない**可能性 |
| **ambient**: `shake` / `bounce` / `float` / `jiggle` | プレビュー CSS translate、エクスポート overlay xy expression。位相や周期が完全一致しない可能性 |
| **キーフレーム**: opacity の補間 | 既知制限: エクスポート時の opacity キーフレーム補間は**未対応**（CLAUDE.md 記載）。プレビューだけ動く |
| **テキスト演出**: `highlight-bar` / `underline-sweep` / `outline-reveal` / `shadow-drop` | 旧 HANDOFF に「FFmpeg 側未対応、プレビューのみ」と明記。エクスポートでは出ない |
| **テキスト演出**: `typewriter` / `stagger-fade` / `wave` / `color-shift` (char) | `composeAnimatedTextLayerVideo` で透過 .mov に焼く経路。プレビューと同じ JS 関数で描画するので**ほぼ一致するはず**だが要確認 |
| **テキスト演出**: `word-pop` / `keyword-color` / `slide-stack` / `zoom-talk` (kinetic) | 同上、同経路 |
| **画像 cover フィット**: `image` レイヤー | プレビュー `object-fit: cover` 相当（overflow hidden + img scaling）。エクスポートは Canvas で `Math.max(w/sw, h/sh)`。同じ式のはずだが画像のアスペクトがズレた場合に挙動差が出る可能性 |
| **video レイヤーの crop** | プレビューは crop を分母にした拡大＋ネガオフセット。エクスポートは ffmpeg `crop` フィルタ。境界 1px ズレなどあり得る |
| **video レイヤーの playbackRate** | ★ 直したばかり（PTS オフセット）。**家の PC + 1660 + h264_nvenc** で同じテンプレを焼いて視覚比較するのが残タスク |
| **shape: "arc"** | 今日追加。プレビューは SVG path、エクスポートは Canvas `arc()` + path 描画 → PNG 焼き込み。**実機テスト未実施**（`templates/test_arc_donut.json` あり） |

## 9. このセッションで得た学び（メタ）

- 「**プレビュー側を直すならエクスポート側も同時に直す**」は CLAUDE.md で明示されているが、`scale=eval=frame` のように **ffmpeg 側の挙動が JS 側と完全に同じにならない場合がある**。今回の `pop` の overshoot 抑制（`min(1)` クランプ）はその例
- ffmpeg の `overlay` framesync は静的 PNG ループの変動サイズフレームに弱い → `pad` で固定サイズ化が必要
- video PTS は overlay の enable window と「絶対 PTS で一致させる」必要がある（`setpts=PTS-STARTPTS` だけだとズレる）→ `setpts=PTS+startSec/TB` を入れる
- Windows の小さい pipe buffer 制限（~4KB）で長時間エンコードがハングする → stdout/stderr を別スレッドで drain する
- argv 32KB 制限 → static は `movie=` フィルタで filter script ファイル経由にして argv を肥大化させない
- ファイル名 ≠ 内部 id の不整合は外部ツール（curio-gen）が原因で生まれることがある → app 側は「id 走査で削除/掃除」する防御を入れる
