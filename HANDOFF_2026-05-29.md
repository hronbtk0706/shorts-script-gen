## 3. ffmpeg export 高速化トライ

### 並列 overlay (filter_complex tree) UI 化
- 既存実装の `SSG_FILTER_GROUPS` env 切替を **ExportModal の select で UI 化**
- 1 (直列) / 2 / 4 / 8 グループから選択可能
- Rust 側で `filter_groups: Option<i32>` payload を受け取り env より優先
- 結果: **CPU 使用率は 32% で頭打ち**（i7-10700 16 スレッドの 5 本相当）→ filter_complex の framesync 待ちがボトルネックで並列化が効かないと判明

### 宝くじテンプレ pad エラー fix（既存ffmpeg 経路）
- `Padded dimensions cannot be smaller than input dimensions` で停止していた
- 原因: **odd-width PNG (1229px 等) が `format=yuva420p` の chroma subsample で実フレーム幅が 1px 増える**
- 修正: pad 目標を `(layer.w_px + 1) & !1` で **偶数に切り上げ**（chroma 補正分の余白確保）
- ついでに `has_dynamic_rotation || has_rotation_static` 時は pad をスキップする保険も追加（rotation の hypot 拡大に対応）

### 結論
- ffmpeg 経路の最適化は **頭打ち**（CPU 律速ではなく filter chain の直列性律速）
- → **構造変化 (WebCodecs) に舵を切る**

## 4. 🌟 WebCodecs エクスポート経路（新規）

### 経路の選定
- **D (Rust 自前 compositor)** は preview ↔ export の mismatch を新規に生むので NG
- **C (WebCodecs 全フレーム焼き)** を採用 — preview コードを再利用 → mismatch 監査が原理的に不要

### 実装方針
```
[毎フレーム]
  preview と同じ Canvas 合成コード (renderLayersOnContext)
    → mediabunny の CanvasSource で h264 encode (Mp4OutputFormat)
[音声]
  OfflineAudioContext で全 audio layer をミックスダウン
    → mediabunny の AudioBufferSource で AAC encode
[動画]
  mediabunny の VideoSampleSink で順次デコード
    → HTMLVideoElement seek 経路 (GOP 遡り) より圧倒的に速い
[保存]
  8MB チャンクで Tauri invoke (JSON IPC の Array.from 制限を回避)
```

### 達成
- **宝くじテンプレ (374 sec, 362 layer) を 約 3 分でエクスポート完了**（既存 ffmpeg 経路: 約 90 分）
- 既存の Live2D キャラ経路 (composeCharacterLayerVideo) と同じ mediabunny ベース

## 7. 注意点・既知の落とし穴

### Tauri IPC の Uint8Array 制限
- `invoke()` は引数を JSON.stringify する。Uint8Array は `Array.from()` で展開される
- → **数百 MB を一度に送ると `RangeError: Invalid array length` で死ぬ**
- 対策: **8MB チャンクで append 保存**（save_final_video の `append: bool` フラグ）

### Tauri release ビルドは DevTools 無効
- F12 押しても効かない（dev のみ）
- → エラーは ExportModal の **errorMsg にスタック付きで UI 表示**するように修正済み

### VideoFrame のサイズプロパティ
- HTMLVideoElement: `videoWidth` / `videoHeight`
- HTMLImageElement: `naturalWidth` / `naturalHeight`
- OffscreenCanvas: `width` / `height`
- **VideoFrame**: `displayWidth` / `displayHeight` （or `codedWidth` / `codedHeight`）← 別系統
- `drawLayer` の video frame 描画で **すべて見る** フォールバック実装にしてある

### 入退場アニメ等の数式
- `src/lib/layerAnimCanvas.ts` (Canvas 用) と preview の `computeLayerAnimStyle` (CSS 用) で**式を完全一致させる**
- 変更時は **両方を必ず同時に更新する**（CLAUDE.md の鉄則）
